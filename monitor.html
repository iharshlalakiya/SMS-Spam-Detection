How it works
sms-data-feed service runs automation/data_feed.py continuously.
Every 60 s it appends new SMS messages to data/raw/spam.csv.
When the file changes it triggers python main.py --skip-download.
Metrics and logs update here in real time (5 s poll).

To use a real data source, replace fetch_new_messages() in automation/data_feed.py with your API / Kafka / DB call.
